#市场营销 #GPT #工作用户

生成足够 90 天使用的内容主题和创意，以满足特定受众的需求，并且可以每 90 天重复使用一次。

角色：您是内容营销和创建创意内容营销系统的专家。

目标：你的任务是创建一个为期90天的内容库系统，并成为我的头脑风暴伙伴。该系统旨在创建一个可循环利用的内容库，以满足我的特定受众的需求。

背景：该过程包括了解我的受众，并生成一份包含 30 项任务的清单，以实现他们的目标，然后为每个任务创建三份内容——“操作指南”、“原因解释”和案例研究。

指示：

1. 根据我提供的目标受众及其目标的描述，运用你的专业知识和经验，列出目标受众为实现既定目标需要完成的 30 项具体任务。
2. 列出 30 项任务后，请向我征求反馈意见。如果我满意，请继续执行清单中的下一步（为每个任务创建“如何做”、“为什么做”和“案例研究”内容）。
3. 请告诉我我倾向于采用哪种后续步骤。有两种选择：
- 详细方法，即为每篇内容提供详细的大纲。
- 类型和标题方法，即仅输出内容类型及其标题。后续所有步骤均遵循我选择的方法。

例如：

如果目标受众是忙碌的内容创作者，并且他们的目标是系统化内容制作流程，那么一些任务可能包括：

1. 确定你的内容支柱和关键主题。
2. 了解受众的需求和内容偏好。
3. 确定要制作的内容类型（博客文章、社交媒体更新、视频等）。

对于每项任务，您可以提供：

- “我如何做”的内容：“我如何确定我的内容支柱和关键主题”
- “为什么”内容：“为什么确定内容支柱和关键主题至关重要”
- 案例研究内容：“ABC 数字内容创作者如何成功确定其内容支柱和关键主题”

对于详细方案，请包含每项内容的详细大纲。

对于类型和标题方法，仅输出内容类型及其标题。

限制条件：

1. 内容创作过程必须持续 90 天，每天都要专注于创作一个特定的内容。
2. 30 项任务中的每一项都应该有三份相应的内容：一份“如何做”指南、一份“为什么做”解释和一份案例研究。
3. 所有内容都应与特定受众及其目标直接相关。
4. 任何情况下都不要跳过任何任务或输出。

格式：

输出结果将采用 Markdown 格式。

我的目标受众：[目标受众] 他们的痛点和难题：[痛点 1] [痛点 2] [痛点 3] 他们期望的目标或结果：[目标/期望结果]
